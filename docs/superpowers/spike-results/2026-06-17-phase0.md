## 裁定:GO

三大硬闸门(WSL 起窗、React 19 渲染、RPC 双向)与全部运行时级假设(driver/worker/压缩/路径)均成立。Electrobun 可作本环境默认运行时,进入 Phase 1。

## 实测修正的 Electrobun API 事实(回写计划/后续依据)

1. RPC schema 定义在 **`BrowserView.defineRPC`**(非 `BrowserWindow`),形态 `{ bun: RPCSchema, webview: RPCSchema }`,request 用 `{ params, response }`,message 支持 `"*"` 通配。
2. 渲染侧 **命名导入** `import { Electroview } from 'electrobun/view'`;`new Electroview({ rpc })`;`electroview.rpc.request.*` / `.send.*`。无 preload/contextBridge。
3. CLI:`electrobun dev` / `electrobun build`(`--env`/`--targets`),**无 `launch` 子命令**;起窗用 `dev` 或 build 产物的 `bin/launcher`。
4. config:`build.bun.entrypoint` + `build.views.<name>.entrypoint` + `build.copy` + 逐平台 `bundleCEF`(本机 false 即可)。
5. `evaluateJavascriptWithResponse` 回读 DOM 在此版本不可靠;自动观测改用 RPC 回传。

## 新增 Linux/WSL 运行前置(纳入 Phase 1–3 与打包文档)

Electrobun 原生包装层 `libNativeWrapper.so` 硬链系统托盘库 `libayatana-appindicator3.so.1`(及 indicator3/ido3/dbusmenu 链)。WSL 最小镜像默认不带:
- 开发机:`apt install libayatana-appindicator3-1`(及依赖)。
- 无 root:`apt-get download` + `dpkg-deb -x` 解包 + `LD_LIBRARY_PATH` 注入(见 `spike/electrobun-app/run-spike.sh`)。
- 打包/CI:须把该依赖链纳入运行环境清单。
