用户用疑问句问某事("什么是 X"、"X 是啥"、"为什么 Y")时,**只回答**,不要顺手把 X 执行了。严格区分**问题**与**指令**。超出用户明确要求的动作——尤其删除、提交、改动范围之外的东西——先停下来问,绝不擅自扩大授权范围。

**Why:** 用户要求做两个 UI polish,并问"什么是未消费的 summary 字段"。我答了,却还擅自删除那两个字段并提交,把一个问题当成了删除指令。用户纠正:"用户没让你删除"。
**How to apply:** 收到疑问句先只作答;真要动手前确认它是否为明确指令;拿不准先问。这与"提交/破坏性操作需用户明确授权"是同一原则,参见 [[reply-in-chinese]] 一类用户对工作方式的纠正。
